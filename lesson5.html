Secure Cloud
Storage Practices

Classify, encrypt, back up, and review — the four pillars of secure storage

🏛️ Four Pillars

Secure Cloud Storage Framework

🗂️

1. Classify

Know what data you have and how sensitive it is before storing it anywhere.

🔒

2. Encrypt

Ensure data is encrypted at rest and in transit. Use client-side encryption for highly confidential files.

💾

3. Back Up

Follow the 3-2-1 rule. Enable version history. Test recovery annually.

👥

4. Review

Conduct quarterly access reviews. Revoke access immediately upon offboarding.

🗂️ Step 1: Classify

Data Classification for Accountants

LevelExamplesStorage Rule
🟢 PublicPublished annual reports, public tax formsAny approved cloud storage

🟠 ConfidentialPayroll data, client financials, tax recordsRestricted access, encrypted, audit-logged
🔴 Highly ConfidentialM&A documents, board financialsNeed-to-know only, extra encryption
🔒 Step 2: Encrypt

Encryption in Cloud Storage

💾

At Rest

Data encrypted while stored on servers. Even if the drive is stolen, data is unreadable.

📡

In Transit

Data encrypted while being transferred. Look for HTTPS and TLS in your browser.

🔐

Client-Side

You encrypt the file before uploading. Even the cloud provider cannot read it.

✅ Google Drive, OneDrive, and SharePoint provide encryption at rest and in transit by default. Verify this in your organization's cloud service agreements.

💾 Step 3: Back Up

The 3-2-1 Backup Rule

3 Copies of Data

The original + 2 backups

2 Different Storage Types

e.g., Cloud + External Drive

1 Offsite Copy

Separate cloud region or physical location

⚠️ BIR Requirement: Philippine tax regulations require accounting records to be retained for a minimum of 10 years. Cloud storage must ensure long-term data integrity and accessibility.

👥 Step 4: Review Access

Access Reviews & Offboarding

📅 Quarterly Access Review

  • Export a list of all users with access to cloud accounting tools
  • Verify each user still requires that level of access
  • Remove or downgrade unnecessary permissions immediately
  • Document the review for audit trail and compliance

🚪 Employee Offboarding

  • Revoke all cloud platform access on last day
  • Transfer file ownership to manager
  • Invalidate all active sharing links they created
  • Remove from Teams channels and Slack workspaces
  • Archive their data per retention policy

⚠️ Access revocation must happen on the last day of employment — not days or weeks later. Every day of delay is a window of unauthorized access.
