Part 4.2 · Hour-by-Hour
A Breach Unfolds: Hour by Hour
A realistic timeline of a phishing-based breach at an accounting firm. Notice where YOU could have stopped it.
1
Hour 0 — Initial Compromise
Accountant clicks phishing link disguised as BIR eFPS notice
Enters credentials on fake login page. Attacker now has valid username + password. No alarm triggered — the login looks legitimate from the attacker's browser.
2
Hour 1–2 — Persistence & Reconnaissance
Attacker sets up email forwarding rule + explores file access
Creates a hidden inbox rule forwarding all emails to external address. Browses OneDrive, SharePoint, and shared mailboxes to map what data is available.
🔍 YOU could spot: unfamiliar inbox rules, sign-in from unusual location
3
Hour 3–12 — Privilege Escalation
Attacker finds shared admin credentials in a Teams chat
Searches chat history for "password," "login," "admin." Finds QuickBooks admin credentials shared in a message 6 months ago. Now has access to the entire financial database.
🔍 YOU could spot: unexpected admin account activity, new sign-in on QuickBooks
4
Day 2–5 — Data Exfiltration
Attacker downloads client tax returns, payroll files, bank details
Uses OneDrive sync to download 2,400 files over 3 days. Small batches to avoid triggering data loss prevention alerts. Also reads email correspondence to prepare BEC attacks.
🔍 YOU could spot: unusual file download activity, sharing permission changes
5
Day 6+ — Monetization
Attacker sends BEC emails to clients requesting payment redirects
Using knowledge from stolen emails, sends convincing payment redirect requests to 12 clients. 3 clients transfer a combined ₱4.8M to attacker's accounts before anyone notices.
🔍 Clients report: "Your firm asked us to change bank details"
Part 4.3 · Attacker Lifecycle
The 6 Stages of an Attack
Based on the MITRE ATT&CK framework — simplified for accountants. Each stage has detectable signs if you know what to look for.
🎣
Stage 1
Initial Access
Phishing email, stolen credentials, exploited vulnerability. The attacker gets their first foothold.
🔍 Detect: Suspicious emails, unexpected MFA prompts
🔧
Stage 2
Persistence
Email forwarding rules, backdoor accounts, scheduled tasks. Ensures access survives password changes.
🔍 Detect: New inbox rules, unknown admin accounts
🔑
Stage 3
Privilege Escalation
Finds admin credentials, exploits misconfigurations, moves from user to admin access.
🔍 Detect: Unusual admin activity, access to restricted systems
🗺️
Stage 4
Discovery & Lateral Movement
Maps the network, finds valuable data, moves between systems (email → file server → accounting software).
🔍 Detect: Unusual file access patterns, cross-system logins
📦
Stage 5
Data Exfiltration
Downloads, copies, or transfers sensitive data out of the organization. Often done in small batches to avoid detection.
🔍 Detect: Large downloads, sharing changes, sync activity
💰
Stage 6
Impact / Monetization
Ransomware deployment, BEC fraud, selling stolen data on dark web, identity theft using stolen TINs.
🔍 Detect: Client reports, unauthorized transactions, ransom notes
🛡️ The earlier in this lifecycle you detect and report, the less damage occurs. Stages 1–2 = minor incident. Stages 5–6 = catastrophic breach. Your vigilance at Stage 1 prevents Stage 6.
