📤 Oversharing
Wrong vs. Right Way to Share
Convenience is not a valid reason to overshare sensitive accounting data
❌ Oversharing (Wrong)
📁 Sharing entire Finance folder
🌐 "Anyone with the link"
♾️ No expiration date set
✏️ Editor access when view-only needed
👥 Entire department added
⚠️ Data Privacy Act Violation Risk
VS
✅ Correct Sharing (Right)
📄 Share only the specific file needed
👤 Specific email addresses only
📅 7–30 day expiration date set
👁️ Viewer access by default
🎯 Only named individuals added
✅ Principle of Least Privilege Applied
🚩 Red Flags
Signs of Oversharing in Your Team
Watch for these warning signs in your daily accounting work
📁
Sharing an entire folder when only one file is needed
🌐
Using "Anyone with the link" for internal documents
♾️
Sending sharing links via email without expiration dates
👥
Adding people to a project channel after the project ended
🚪
Keeping former employees or contractors in shared workspaces
✏️
Using Editor access when Viewer would be sufficient

⚠️ Oversharing is often accidental — done with good intentions. But the legal consequences are the same whether intentional or not under the Data Privacy Act.

⚙️ Misconfiguration
Common Mistakes & How to Fix Them
Misconfiguration is the #1 cause of cloud data breaches worldwide
⚙️ Misconfiguration🔴 Risk✅ Fix
Public SharePoint siteInternet can access all filesSet to Private; require authentication
"Anyone with link" on DriveUncontrolled access spreadChange to "Specific people" only
No MFA on accounting softwareAccount takeover via stolen passwordEnable Multi-Factor Authentication
Shared team login credentialsNo audit trail, accountability gapCreate individual user accounts
Inactive accounts not removedFormer employees retain accessConduct quarterly access reviews
No link expiration datesPermanent unauthorized accessSet 7–30 day expiration on all links
Overpermissive API integrationsThird-party apps access too much dataReview and restrict app permissions
🔑 Critical Control
Multi-Factor Authentication (MFA)
MFA stops 99.9% of automated account attacks — enable it on everything
🔑
Factor 1
Something You Know
Password or PIN
+
📱
Factor 2
Something You Have
Authenticator app, SMS code, hardware token
+
👁️
Factor 3
Something You Are
Fingerprint, face recognition, biometrics
=
✅
Secure Access Granted
Even if password is stolen, attacker cannot log in without the second factor

❌ Without MFA: A stolen password gives an attacker complete access to QuickBooks Online, SharePoint, and all financial records — instantly.

📋 Monthly Audit
Misconfiguration Audit Checklist
Click each item to mark complete — run this monthly for your accounting team
Are all sharing links set to expire?
Are former employees/contractors removed from all platforms?
Are sensitive channels/folders set to private?
Is MFA enabled for all users on all platforms?
Are permission levels reviewed and appropriate?
Are third-party app integrations reviewed?
Is there a log/audit trail of who accessed what?
Are files labeled with appropriate sensitivity levels?
