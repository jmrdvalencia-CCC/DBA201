⚠️ Risks 1–4 Deep Dive
Access, Sharing, Config & Shadow IT
🔓
1. Unauthorized Access
Occurs when people who should NOT have access can view, edit, or download data — caused by expired links never removed or unrevoked accounts.
📌 A shared OneDrive link set to "Anyone with the link" lets a former employee access confidential financial reports months after leaving.
📤
2. Oversharing
Giving more access than necessary — sharing an entire folder when only one file is needed exposes all other files to unintended viewers.
📌 Sharing the entire "Finance 2024" folder with an external auditor who only needed the Q3 income statement.
⚙️
3. Misconfiguration
Incorrect settings expose data unintentionally. This is the #1 cause of cloud data breaches worldwide — not hackers, but insiders setting things up wrong.
📌 A SharePoint site set to "Public" during migration makes internal salary data visible to the entire organization.
👤
4. Shadow IT
Using unauthorized cloud tools bypasses all organizational security controls, encryption standards, and compliance requirements.
📌 An accountant uploads payroll files to personal iCloud to work from home — bypassing all company security controls.
⚠️ Risks 5–7 Deep Dive
Credentials, Links & Data Loss
🔑
5. Weak / Shared Credentials
Sharing one login among multiple accountants destroys the audit trail, creates a single point of failure, and violates most SaaS platform terms of service.
📌 3 accountants share one QuickBooks Online login — impossible to track who made which journal entry.
🔗
6. Insecure File Sharing Links
Links without expiration dates or password protection remain permanently active. Anyone who receives or finds the link can access the document indefinitely.
📌 A financial statement link shared in Slack 2 years ago is still active — anyone with the link can still open it today.
🗑️
7. Data Loss / Accidental Deletion
Cloud tools allow multiple users to edit or delete files simultaneously. Without version history enabled, permanently deleted tax documents violate BIR's 10-year retention requirement and can result in penalties.
📌 A team member deletes a folder of 5-year-old tax documents thinking it was a duplicate. No version history = gone forever. BIR penalty risk.

📖 Real-World Scenarios B & C
More Incidents in Accounting Firms
These are not hypothetical — these incidents occur regularly
🔓
Scenario B
The Expired Contractor
A freelance bookkeeper's QuickBooks Online and SharePoint access was never revoked after their 3-month contract ended. Six months later, they — now at a competitor — still have full access to all financial records.
🔓 Unauthorized Access
✅ Fix: Revoke access on last day of contract
⚙️
Scenario C
The Public Slack Channel
The accounting team's #invoices Slack channel was accidentally set to public during workspace setup. All interns, new hires, and IT staff can now view confidential client billing data.
⚙️ Misconfiguration
✅ Fix: Set financial channels to Private always
