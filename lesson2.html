⚠️ Risks 1–3

Access, Sharing & Configuration

1

🔓 Unauthorized Access

People who should NOT have access can view, edit, or download data. Caused by links that never expire or unrevoked accounts.

2

📤 Oversharing

Giving more access than necessary — often done for convenience. Sharing an entire folder when only one file is needed.

3

⚙️ Misconfiguration

Incorrect settings that expose data unintentionally. A SharePoint site set to "Public" instead of "Private" is a classic example.

⚠️ Risks 4–7

Credentials, Links & Data Loss

4

👤 Shadow IT

Using unauthorized cloud tools not approved by IT. Uploading payroll files to a personal Google Drive bypasses all security controls.

5

🔑 Weak / Shared Credentials

Sharing one QuickBooks login among three accountants destroys the audit trail and creates a single point of failure.

6

🔗 Insecure File Sharing Links

Links without expiration dates remain active for years. Anyone who finds the link can access the document indefinitely.

7

🗑️ Data Loss / Accidental Deletion

Without version history, a deleted folder of tax documents is gone permanently — violating BIR's 10-year retention requirement.

📖 Real-World Scenario A

The Overshared Payroll Sheet

Scenario A

Maria, an HR-Accounting staff, creates a Google Sheet with all employee salaries. She shares it with "Everyone in the organization" so the Finance VP can view it quickly.

Two weeks later, employees discover their salaries are visible to all 200 staff members. This violates data privacy and causes serious damage to employee trust.

🔴 Risk: Oversharing  |  Fix: Share with specific email addresses only — never "Everyone in the organization" for sensitive data.

📖 Real-World Scenario B

The Expired Contractor's Access

Scenario B

A freelance bookkeeper was given access to a company's QuickBooks Online and SharePoint for a 3-month project. After the contract ended, no one revoked access.

Six months later, the bookkeeper — now working for a competitor — still has full access to all financial records, client data, and internal reports.

🔴 Risk: Unauthorized Access  |  Fix: Revoke all cloud access on the last day of any contract or employment.

📖 Real-World Scenario C

The Public Slack Channel

Scenario C

An accounting team uses a Slack channel called #invoices to share client billing information. The channel was accidentally set to public during workspace setup.

Anyone in the Slack workspace — including interns, new hires, and IT staff — can view confidential client invoice data and billing amounts.

🔴 Risk: Misconfiguration  |  Fix: Always set financial channels to Private. Audit channel visibility settings monthly.
